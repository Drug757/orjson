## Orjson High-Speed Processor
Проект представляет собой систему имитации ETL-процесса (Extract, Transform, Load) для обработки больших массивов данных. Основная цель — демонстрация возможностей библиотеки orjson при работе со сложными структурами данных Python, такими как Dataclasses, UUID, Enum и Datetime.

## Описание системы
Система генерирует синтетические профили пользователей, сериализует их в высокопроизводительный формат NDJSON (Newline Delimited JSON) и выполняет обратное чтение. Использование orjson позволяет достичь скорости обработки, значительно превышающей возможности стандартного модуля json.

## Требования

    Python версии 3.8 или выше.

    Библиотека orjson.


## Установка и запуск

    1. Установите необходимую зависимость:
        pip install orjson

    2. Убедитесь, что все файлы проекта находятся в одной директории:

        models.py — описание структур данных.
        generator.py — логика создания тестовых данных.
        processor.py — инструменты сериализации и работы с файлами.
        main.py — точка входа и замер производительности.

    3. Запустите программу
        python main.py


## Структура проекта

Файл,                   Описание
models.py,              "Определение классов данных (UserProfile, LogEntry) и перечислений (UserRole)."
generator.py,           Функция генерации заданного количества объектов со случайными данными.
processor.py,           Реализация методов записи и чтения NDJSON с использованием флагов orjson.
main.py,                Координация работы модулей и вывод метрик времени выполнения.


## Ключевые особенности реализации

    Производительность: Использование orjson.dumps возвращает байты, что позволяет записывать данные напрямую в файл без дополнительного этапа кодирования в UTF-8.

    Поддержка типов: Включены флаги OPT_SERIALIZE_DATACLASS и OPT_SERIALIZE_UUID для автоматической обработки сложных объектов.

    NDJSON: Флаг OPT_APPEND_NEWLINE обеспечивает автоматическое добавление символа новой строки после каждого объекта, что соответствует стандарту построчного JSON.

    Память: Потоковое чтение файла по строкам позволяет обрабатывать объемы данных, превышающие объем оперативной памяти.

## Результат работы 
    После запуска программа создаст файл large_data.ndjson в корневой директории проекта и выведет в консоль время, затраченное на генерацию, запись и чтение данных.
